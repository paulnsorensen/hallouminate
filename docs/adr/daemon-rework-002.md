# ADR daemon-rework-002: WorkClass typed guards on one DaemonState  [status: accepted]

- **Context:** One undifferentiated `enter_connection` counter + `touch_activity` clock served three unrelated subsystems (client RPCs, watcher batches, maintenance tick) for different reasons. The watcher stamping the shared clock kept the daemon permanently "Active" (2026-07-17), starving maintenance; ADR-002 (maintenance must not stamp the idle clock) was an earlier patch on the same class. The External/Internal boundary the design needs did not exist as a boundary.
- **Decision:** `enter_connection(WorkClass)` / `touch_activity(WorkClass)` with `WorkClass::{External, Internal}` on the single existing `DaemonState`. Idle-exit gates on both classes (preserves ADR daemon-idle-exit-003: no teardown mid-write releasing the flock under a live LanceDB writer). Maintenance-defer gates on External only — housekeeping never defers housekeeping. Classification: server RPC handlers = External; watcher, catch-up, maintenance = Internal.
- **Alternatives rejected:** two separate tracker structs (maximum explicitness, but every background-task author must pick the right tracker — the same forget-a-guard hazard the inventory flagged as tension #3).
- **Consequences:** ADR-002 and ADR daemon-idle-exit-003 both preserved explicitly. A test must assert every `enter_connection` caller declares a class (misclassification re-opens starvation; churn/defer ladders are the defense in depth).
