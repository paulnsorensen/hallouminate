# ADR daemon-rework-001: Writer backpressure with paced forced maintenance  [status: accepted]

- **Context:** ADR pressure-aware-maintenance-001..003 made maintenance defer indefinitely under activity/PSI pressure (lesson of the 2026-07-13 114.7GiB incident: never pile compaction onto a saturated daemon). The 2026-07-17 incident showed the missing bound: internal churn kept the daemon "Active" and maintenance starved for 17h with only a WARN. Mature systems (Postgres anti-wraparound vacuum, RocksDB write stalls, SQLite RESTART checkpoints) all pair best-effort maintenance with a forced mode that cannot be deferred; Lance OSS has no self-triggered path, so the caller owns the guarantee.
- **Decision:** Graduated writer backpressure on real debt (`MaintenanceDebt{fragments, stale_versions}`): Soft ⇒ per-mutation cost delay; Hard ⇒ mutations block (bounded wait, retryable error) and the maintenance run is forced. Hard overrides both the Active defer and the PSI defer, but elevated PSI switches the forced run to paced mode — bounded slices, sleeps between, write guard released between slices. Reads never stall.
- **Alternatives rejected:** throttled-force without backpressure (Postgres shape — user preferred the RocksDB guarantee); defer-count-only bound (quiet-but-growing store still bloats); PSI-supreme (re-opens indefinite deferral through the pressure door).
- **Consequences:** amends pressure-aware-maintenance-001..003 (their lesson survives as pacing, not as unbounded deferral). At Hard, mutations already stalled ⇒ debt stops growing ⇒ paced completion is bounded. Requires #216 (client RPC timeouts) wired — absorbed as a spec dependency.
